This week shifted the focus from interaction design to system-level thinking, particularly through the introduction of design systems and frameworks. While previous weeks emphasised user flows and structure, this stage highlights the importance of consistency and scalability in interface design.

One key concept is that design should not be approached as isolated screens, but as part of a larger system. A design system provides a shared set of components, styles, and rules that ensure consistency across different parts of an application. This includes elements such as buttons, typography, spacing, and layout patterns. By reusing these components, designers can create more coherent and efficient designs.

The use of frameworks, such as Bootstrap, reinforces this idea. Instead of building everything from scratch, frameworks provide predefined structures and components that can be adapted to different use cases. This not only speeds up development but also encourages standardisation. However, this also introduces a trade-off between flexibility and efficiency, as relying too heavily on frameworks can limit creative control.

Another important takeaway is the role of constraints in design. Rather than being restrictive, constraints can guide better decision-making by narrowing down possibilities. For example, using a grid system helps maintain alignment and visual hierarchy, making interfaces easier to understand and navigate.

This week also made me realise that consistency is not just a visual concern, but a usability issue. When users encounter familiar patterns, they can navigate a system more easily without needing to relearn interactions. This reduces cognitive load and improves the overall user experience.

Overall, this week reinforced the idea that good design is systematic rather than ad hoc. By thinking in terms of reusable components and structured frameworks, designers can create interfaces that are not only visually consistent but also scalable and maintainable over time.
